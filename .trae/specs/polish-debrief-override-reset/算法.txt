## 美团校园 AI Hackathon – 配送分配自主求解器 最终方案

### 1. 总体定位：双层分离、内核硬化、自演进闭环

本方案构建的 **AutoResearch Agent** 不再仅仅是一个比赛提交物，而是一个 **“离线自主科研 + 在线稳定推理”双层系统**。人类开发者彻底退出“写算法”的微观层面，转而定义智能体的研究边界与基础组件库。智能体在离线阶段通过数千轮自主实验，从丰富的算法原语中筛选并组合出最优求解策略；在线阶段仅部署经过硬化的最优代码模板与轻量级策略路由，在 2–5 秒限时内稳定输出高质量分配方案。

**核心突破：**
- **范式跃迁**：从“手写启发式”到“AI 自研求解器”。
- **工程硬化**：引入 **Primitive Library**，杜绝 LLM 凭空生码带来的不稳定性。
- **双环分离**：离线进化算法结构与参数，在线仅做轻量推理与策略选择。
- **深度反思**：从简单数值升降升级为**归因分析与结构优化**。
- **工业真实约束**：多目标效用动态加权，融入出餐、骑手意愿、风险区域等物理扰动。

### 3. 核心模块深度设计

#### 3.1 算法原语库 (Primitive Library)
这是整个系统工程稳定性的基石。LLM 不再从零生成代码，而是在预定义的高质量原子组件上进行**选择、组合与参数调优**。

**基础层 (稳定与快速)**
- `GreedyMinCost`：仅按成本排序贪心分配。
- `GreedyMaxWillingness`：按意愿度排序贪心。
- `ConflictAwareGreedy`：考虑任务重叠度的贪心，避免小单抢占大单。
- `UtilityDensityGreedy`：使用多目标效用密度 $\frac{\alpha |T| - \beta s + \gamma w}{\text{时间估计}}$ 排序。

**中级层 (约束处理与松弛)**
- `LagrangianRelaxation`：拉格朗日松弛处理骑手唯一性硬约束。
- `BipartiteMatchingSparse`：将问题转化为二分图最大权匹配，对密集候选组稀疏化。
- `LocalSearch_2opt`：对已有解进行 2-opt 或 Swap 操作，消除冲突并改进。
- `BeamSearch`：宽度受限的束搜索，平衡探索与耗时。

**高级层 (近似与分解)**
- `ILP_WarmStart`：利用贪心解作为初始解，调用 `ortools` 或 `pulp` 求解小规模整数规划，超时则输出当前最优。
- `GraphClusteringDecomp`：根据任务-骑手图聚类分解问题，降维求解。
- `AdaptiveNeighborhoodSearch`：自适应大邻域搜索，动态调整破坏/修复力度。
- `MonteCarloRollout`：对不确定性（意愿波动）进行蒙特卡洛采样，选择最鲁棒分配。

**LLM 的职责明确为：**
1. 阅读当前日志与历史表现，提出 **“策略组合假设”**（如：先用 `GraphClusteringDecomp` 分解，再在每个簇内使用 `ILP_WarmStart`）。
2. 指定每个组件的**超参数**（如束宽、迭代次数、惩罚系数）。
3. 生成安全的 **Python 胶水代码** 将组件拼接，不再编写算法核心逻辑。

#### 3.2 深度反思引擎 (Semantic Reflection)
替换简单的 if-else 效用比较，建立真正的科研归因链路。

**反思维度：**
- **冲突归因**：分析被拒绝的分配中，哪些任务冲突频繁，是否存在图结构中的“瓶颈任务”。
- **耗时分析**：当耗时超过阈值（如 4 秒）时，定位耗时组件，建议简化或替换（如 ILP 改贪心）。
- **效用成分拆解**：分别追踪履约单量、总成本、平均意愿度的变化，判断效用提升的正负向驱动力。
- **分布漂移检测**：当前 `large_seed` 的订单-骑手图密度、意愿均值和方差与历史成功案例的差异，建议策略调整方向。

**实现示例 (Prompt 片段给 LLM)：**
> “本次评估结果：效用 15234，耗时 3.2s。相比最优记录，成本上升了 8%，但意愿度提升了 12%，净效用增加。耗时中 `ILP_WarmStart` 占用 2.1s，存在超时风险。冲突分析显示 task_id ‘T0345’ 在 60% 的失败分配中出现，疑似高重叠度任务。请提出两种不同的策略改进方案，重点降低耗时或缓解‘T0345’瓶颈。”

#### 3.3 多目标动态效用函数
不再使用固定的 $\alpha, \beta, \gamma$，而是由离线 Agent 在训练中**自主探索最佳权重分布**，并可针对不同数据特征（如配送高峰、低意愿时段）动态调整。

**动态效用公式：**
$$\mathcal{U} = \sum_{i \in \mathcal{N}} \left( \alpha(s) \cdot |T_i| - \beta(s) \cdot s_i + \gamma(s) \cdot w_i + \delta \cdot R_i \right)$$
其中 $s$ 为数据特征向量（订单密度、平均意愿等），$R_i$ 为骑手风险溢价（基于历史卡餐/差评率），$\delta$ 为风险惩罚系数。Agent 会在离线阶段通过网格搜索或贝叶斯优化确定 $\alpha(s), \beta(s), \gamma(s), \delta$ 的最优映射。

#### 3.4 离线演进与在线部署分离
**离线阶段 (Training)**
- 输入：多个历史 `large_seed` 数据集（真实脱敏 + 合成场景）。
- 过程：AutoResearch Agent 执行数千次“生成-评估-反思”循环，不断进化。
- 输出：
  1. **Top-K 策略模板**（如 K=5，包含不同组合）。
  2. **最优参数集**（权重、束宽、迭代次数等）。
  3. **策略路由器**：一个轻量级模型/规则，根据输入特征选择最佳模板。

**在线阶段 (Inference)**
- 提交的 `solver.py` 结构：
  ```python
  def solve(input_text):
      features = extract_features(input_text)   # 密度、意愿分布等
      strategy_id = router(features)            # 离线训练得到的路由器
      if strategy_id == 0: return template_A(input_text)
      elif strategy_id == 1: return template_B(input_text)
      ...
      else: return fallback_greedy(input_text)  # 绝对安全兜底
  ```
- 所有被选中的模板均经过离线充分验证，保证单次调用 ≤ 5 秒，且结果严格无冲突。
- 路由器本身极轻量（决策树或简单规则），耗时 < 0.1 秒。

### 4. 沙盒评估与稳定性保障

- **硬时限控制**：在线沙盒使用 `subprocess` 强制 timeout=4.8s，预留余量。
- **冲突强制校验**：每次评估后自动扫描分配结果，若出现重复骑手或任务，立即返回极大负效用并标记为无效解。
- **内存溢出保护**：评估进程资源限制（`resource` 模块），防止恶意或错误代码撑爆内存。
- **数据空值/异常兜底**：所有求解器模板均需包含对空输入、单骑手等边界情况的处理逻辑，作为组件库的强制规范。

### 5. 工业级自适应扩展 (保留你的前瞻性内容)

虽然在线提交不执行这些，但离线 Agent 的演进已经考虑以下现实扰动，使得生成策略具备强鲁棒性：
- **出餐联动**：Agent 在历史数据中识别“备餐时长”字段，自动为未出餐任务施加延迟分配惩罚。
- **风险区域**：根据骑手历史交付失败率构建“风险热区”，在效用中增加区域风险折扣，促使求解器避免将高危订单合并进大任务组。
- **主动改派模拟**：在沙盒中模拟 5% 的随机订单取消或骑手故障，评估解的二次调整代价，筛选出“高弹性”分配策略。
